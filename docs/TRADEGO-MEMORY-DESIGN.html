Trade2GO 记忆系统设计 · C′ 方案

外贸跟单(CheckToGo.ai = Trade2GO 别名) ｜ 2026-06-01 ｜ 待你确认 → 落 spec → 实现
一句话:用强模型(Sonnet)当"老师"把对话里搞定的口径/习惯结晶成纯文本规则并冻结版本,弱模型(Haiku 4.5)当"执行工"照规则答。确定的不推翻、知识单调向前、成果可复用到别的模型。订单进度走结构化状态表,语义记忆走 Claude memory tool。

0 · 怎么走到 C′ 的(决策链)

取消打钩沉淀→ AI 自主从对话判断记/更新→ 选 C:Claude memory tool→ ⚠️ OpenRouter 不透传 memory beta(400)→ C′:tradego 直连 Anthropic

验证依据:OpenRouter 的 Claude 端点不支持 context-management-2025-06-27 beta,带该 header 直接 400 → memory tool 必须直连 Anthropic。国产模型可移植性本期不做(C′ 绑 Claude 家族,够 Sonnet↔Haiku 4.5 切)。

1 · 总体:三种存储,各管一摊

存什么怎么存为什么
订单 / 跟单进度(有状态)Supabase 结构化表 tradego_orders(status 枚举 + 双时序 valid_from/to)+ 专用工具状态机绝不能塞自由文本(调研头号坑:会自相矛盾)
冻结规则 / 候选 / 客户事实(语义)Claude memory tool(/memories/… 文件,后端落 Supabase)原生 view/create/str_replace,省脚手架 + context editing 省 token
行业基线 prompt代码 INDUSTRY_PROMPTS['外贸']不变的底座

memory 文件约定:rules/frozen.md(冻结,带版本号)· rules/candidates.md(候选)· customers/<id>.md(客户长期事实)。按 expert_id + product 隔离。

2 · 写入 & 冻结(防推翻的核心)

热路径(边聊边记)

Sonnet 边回答边判断,新东西只写进 candidates.md(不动冻结层)。廉价关键词门控:出现 金额/单号/FOB·CIF/已发货… 才触发判定,寒暄直接跳过省 token。判定逻辑借 mem0 的 ADD / UPDATE / NOOP prompt(让 LLM 判,不写 if-else)。

候选 → 冻结:三种信号(全要)

  1. 大咖一句话确认 —— AI 识别「对 / 以后都这样」→ 候选升冻结 + 版本号 +1(替代旧的打钩按钮)
  2. 重复自判 —— 同口径连续被遵循 N 次且无反驳 → 后台 Sonnet 巡检自动升级
  3. 版本快照 + 测评集门槛 —— 攒一批候选 → 跑外贸场景测评集 → 过了才 cut 成 frozen.md vN

防推翻

frozen.md 只能由上面三条确认/版本路径改;执行模型被明确告知冻结规则是权威、不许私自覆盖。要改先经确认或新版本。

3 · 订单状态机

报价→ 待 PI→ 已付定金→ 生产中→ 已发货→ 收尾

tradego_orders:status 枚举 + 金额/币种/客户 + 双时序(状态变更不删旧值,旧行置 valid_to,留时间线)。工具:update_order_status(只允许合法迁移)+ query_orders。

⚠️ 待你确认:上面枚举要不要按你真实跟单流程改。

4 · 检索 · 注入 · 模型路由

注入(system prompt 分层)

  • 行业基线 prompt
  • 冻结规则(权威,prompt-cached 5min)
  • 当前活跃订单摘要(查表)
  • 相关候选;更深回忆走 memory tool view

模型路由(product=tradego → 直连 Anthropic)

  • 现在:Sonnet 4.6 = 老师(写规则)
  • 规则稳了:回答切 Haiku 4.5(执行,约 1/3 成本)
  • 记忆固化仍留 Sonnet 后台巡检

省钱三件套:prompt caching(规则当缓存前缀)+ context editing(长会话清旧 tool 结果)+ 关键词门控判定。

5 · 测评集(一举两得)

攒一组真实跟单对话 + 期望行为:① 冻结新版本前的门槛 ② 以后换 Haiku / 国产模型时拿它验证「成果可复用」 —— 正好兑现你要的可移植验证。

6 · 分阶段落地

阶段做什么
P0tradego_orders 表 + update/query 工具 + 冻结规则纯文本注入(手动种几条)→ 跑通最小闭环
P1memory tool 后端(Supabase)+ Sonnet 写候选 + 大咖一句话确认冻结
P2后台巡检(重复 N 次升级)+ 版本快照 + 测评集门槛
P3回答切 Haiku 4.5 + context editing;退休旧 trg_save_memory_on_expert_up 打钩触发器

7 · 调研背书 & 避坑

抄进来的范式

  • mem0 A.U.D.N. 判定(何时记/更新)
  • Zep/Graphiti 双时序(订单不删旧值)
  • Generative-Agents 三因子检索(recency+importance+相关性)

避的坑

  • 订单进度不用自由文本(会自相矛盾)→ 结构化表
  • 高置信陈旧记忆:大咖一句话可即时 invalidate + cron 审计
  • 判定器漏写:关键词门控 + 系统提示反复强调"做了决策就记" + 小样本回归

  2. "重复 N 次"自动冻结的 N 取几?(建议 3)
  3. P0 范围够不够小?要不要先只做 P0 验证体验,再继续?
