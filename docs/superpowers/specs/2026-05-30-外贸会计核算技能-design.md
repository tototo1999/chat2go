# 外贸会计 / 核算技能 — 设计 (子项目②)

日期: 2026-05-30
产品: Trade2GO.ai (chat2go.xyz, repo tototo1999/tradego)
后端: chat2go-worker (Modal, worker/chat2go_worker.py)

## 背景

cutover 前 Hermes 时代有 `contract_lib.py` / `excel_lib.py` 等外贸专属能力,2026-05-26 全 cloud cutover 后丢失(worker INDUSTRY_ALIASES 注释明确写"外贸跟单老房复用外贸 prompt;失去 contract_lib 等 Hermes 专属能力")。本子项目重建会计核算能力,以 serverless 原生方式(Claude tool-use)实现。

## 目标

给 Trade2GO 的 AI 加上**精确的外贸会计核算能力**,覆盖四域:
1. 成本 + 报价 + 利润闭环
2. 汇率结算 / 多币种
3. 对账 / 账期
4. 出口退税 + 佣金

核心要求: **金额由 Python 精确计算(Decimal),不让 LLM 心算**,结果可审计。

## 架构

### tool-use 计算循环 (替代当前单次调用)

当前 `ingest()` 是一问一答单次 `messages.create`。改为循环:
1. AI 收到用户问题 → 判断需要算账 → 申请调用某计算工具 (tool_use block)
2. worker 用对应 Python 函数精确计算 → 把结果作为 tool_result 回灌
3. 循环直到 AI 产出最终自然语言 + markdown 表格回复
4. UPDATE placeholder

约束:
- **作用域隔离**: 仅当 `industry ∈ {外贸, 外贸跟单}` 时挂载这些 tools。其它 6 产品 (speak2go/well2go/essay/korean/chat2go算命/...) 完全不受影响,走原单次路径。
- **循环上限**: 单轮最多 5 次工具调用 (MAX_TOOL_ITERS=5),防失控/防成本爆炸。超限则用已有结果让 AI 收尾。
- system prompt: 外贸 prompt 增补会计专业知识 + 各工具何时使用的指引。

### 精度规则

- 全部用 `decimal.Decimal`,禁用 float 做金额运算。
- 默认金额保留 **2 位小数, ROUND_HALF_UP** (人民币/美元惯例)。
- 单价 (unit price/unit cost) 可保留 **4 位**。
- 汇率本身 4 位。
- 百分比 (利润率/退税率/增值税率) 入参用小数 (0.13 表示 13%),展示转百分比。

### 汇率来源

v1: **汇率作为入参** (用户或 AI 在对话中给出"今天美元 7.2",或 fx_convert 工具的 rate 参数)。
不接实时汇率 API (留作后续增强,避免外部依赖)。

## 7 个确定性计算工具 (纯函数, 可单测)

每个工具: 明确入参 schema → 返回 dict (含最终值 + 计算明细 breakdown, 供 AI 解释和审计)。

1. **calc_unit_cost** — 单位成本
   - 入参: purchase_total, freight, duty, misc_fees, quantity
   - 返回: unit_cost, total_cost, breakdown
   - 例: 采购100 + 运费20 + 关税15, 数量10 → total=135, unit_cost=13.50

2. **quote_from_margin** — 按目标利润率倒推报价 + 贸易术语换算
   - 入参: unit_cost, target_margin (小数), incoterm (FOB/CIF/CFR), freight?, insurance?
   - 返回: quote_price, 各 incoterm 价, breakdown
   - 报价 = 成本 / (1 - 利润率); CIF = FOB + 运费 + 保费; CFR = FOB + 运费

3. **order_pnl** — 单笔订单损益
   - 入参: revenue, cost, expenses, commission, tax
   - 返回: gross_profit, net_profit, gross_margin, net_margin, breakdown

4. **fx_convert** — 汇率换算 / 多币种统一
   - 入参: amount, from_ccy, to_ccy, rate (或 amounts[] 列表 + rate map)
   - 返回: converted, base_total, breakdown

5. **export_rebate** — 出口退税
   - 入参: purchase_amount (含税), vat_rate (小数), rebate_rate (小数)
   - 返回: rebate_amount, breakdown
   - 退税额 = 采购额 / (1 + vat_rate) × rebate_rate

6. **commission** — 佣金核算 + 净利还原
   - 入参: base_amount, commission_rate (小数) 或 commission_fixed, net_before
   - 返回: commission_amount, net_after, breakdown

7. **reconcile** — 对账 / 账期 aging
   - 入参: receivables[] / payables[] (每条 {amount, due_date, paid}), as_of_date
   - 返回: total_recv, total_pay, balance, aging_buckets (0-30/31-60/61-90/90+), unmatched

## 测试 (TDD)

7 个工具是纯函数 → pytest,已知输入对已知输出,先写测试再写实现。
- 每个工具 ≥ 3 个 case (正常 / 边界 / 精度舍入)。
- 工具全绿后再做 tool-use 联调 (worker loop)。
- 联调验收: 真实从后端触发一条算账问题,看 AI 是否正确调用工具 + 数字精确。

## 范围边界

- **不含**生成 .xlsx/.pdf 文件 (那是子项目③)。本子项目核算结果用**聊天里的 markdown 表格**呈现。
- **不含**实时汇率 API。
- **不含**修改其它 6 个产品的行为 (作用域隔离保证)。

## 部署

- 改 `worker/chat2go_worker.py` → `modal deploy`。
- 工具函数可单独成模块 `worker/trade_accounting.py` 便于单测 (worker import)。
- 无前端改动 (markdown 表格已支持渲染)。
